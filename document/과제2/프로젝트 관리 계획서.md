# 1. 서론 #

## 1-1. 문서 목적 및 범위 ##
<br/>
이 프로젝트 계획서는 **맛집탐방 모임 관리 시스템**에 대한 전반적인 내용을 정리하여 프로젝트를 진행하는 데 있어 각 세부 사항의 지표로 삼는 것을 목적으로 한다. <br/>
<br/>
이 문서는 다음과 같은 범위의 지표로 삼는 것을 목적으로 한다. <br/>
1) 프로젝트 개요 (프로젝트 정의, 주요 기능 정리) <br/>
2) 개발 계획 및 환경 (개발 절차 모형, 내용 및 일정, 소프트웨어 환경) <br/>
3) 품질관리 (산출물 관리) <br/>
<br/>

## 1-2. 프로젝트 개요 ##

### 1-2-1. 프로젝트 정의 ###
링크만 클릭하면 접속할 수 있는 웹사이트를 만들어 맛집탐방 인원모집과 일정관리를 쉽게 할 수 있도록 돕는다. <br/>
<br/>

### 1-2-1. 주요 기능 설명 ###
1) 원하는 식당을 정해서 맛집탐방 참여자를 모집할 수 있는 '레이드' <br/>
2) 먹고싶은 메뉴를 제안하고, 좋아요를 눌러 관심도를 나타낼 수 있는 '먹고싶은거' <br/>
3) 자신의 프리한 시간을 올려 맛집탐방 일정에 데려가 달라는 글을 쓸 수 있는 '파티구직' <br/>
<br/>

## 1-3. 프로젝트 목표 ##
이 프로젝트는 맛집탐방 모임을 쉽게 만들고 참여하도록 하여 아래의 목표를 달성하고자 한다. <br/>
<br/>
1) 사람들에게 맛집탐방 일정을 제안하여 함께할 사람을 모집 <br/>
2) 다른 사람들의 일정 모집을 보고 쉽게 참여할 수 있도록 <br/>
3) 먹고싶은 메뉴를 제안하여 다른 사람들이 일정을 계획하는데 도움 <br/>
<br/>

# 2. 개발 계획 #

## 2-1. 개발 절차 모형 ##
1인 개발로 먼저 현재 정의된 목표만 만들고, 나중에 이용자들의 피드백을 수용하여 웹사이트를 업데이트 할 예정이다. <br/>
일단 전체적인 그림은 잡혀 있기 때문에 점진적 개발로 주요 기능을 하나씩 개발해나가도록 한다. <br/>
<br/>

## 2-2. 개발 활동 ##
프로젝트 요약 : 프로젝트 요약서 작성 <br/>
프로젝트 계획 : 프로젝트 계획서 작성 <br/>
요구사항 정의 : 요구사항 정의서 작성 <br/>
요구사항 분석 : 요구사항 분석서 작성 <br/>
Product Backlog : 요구사항을 우선순위대로 순서화 <br/>

## 2-3. 개발 일정 ##
1일차 : 프로젝트 요약 <br/>
2일차 : 프로젝트 계획 <br/>
