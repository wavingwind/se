## 프로젝트 명칭 ##
맛집탐방 파티원 모집 사이트 개발



## 개발 시스템 명칭 ##
밥항공기



## 소개 ##
맛집탐방 소모임의 멤버들이 가고싶은 가게를 사이트에 올리면 다른 멤버들이 참가신청을 할 수 있는 시스템.



## 계정 ##
1-1. 소모임 멤버들이 회원가입해서 사용할 수 있는 로그인 계정
1-2. 사이트 링크를 클릭하면 바로 사용할 수 있는 게스트 모드

1-3. 회원가입 할 때 유저는 선택적으로 프로필 이미지를 추가할 수 있음. 
1-4. 프로필 이미지는 일정신청글에 댓글을 달거나 건의사항에 글을 올릴 때 표시됨.

고민중인 것
- 회원가입을 그냥 바로 되게 할 지, 승인받아야 회원가입이 되게 할지 (광고성 가입을 막기 위해)
 
## 가게/일정/메뉴 제안 ##
2-1. 로그인한 유저는 자신이 가고싶은 가게나 먹고싶은 메뉴를 제안하는 글을 게시판에 작성할 수 있음.
2-2. 게스트 모드에서는 제안하는 글을 작성할 수 없지만, 다른 유저가 제안한 일정에 자신의 이름을 적고 참가신청할 수 있음.
2-3. 가게(일정)를 신청할 때 '대표메뉴 몇가지', '이미지', '가고싶은이유', '위치', '네이버지도 링크'를 각각 선택적으로 추가할 수 있음.
2-4. 같은 ip의 유저가 하나의 일정에 여러번 참가신청할 수 없음. (게스트 모드로 한사람이 여러번 신청해서 신청인원을 늘리는 것을 막기 위해.)

2-5. 가게나 일정을 신청한 사람들의 명단을 볼 수 있음. 같이가기 싫은 사람이 있으면 언제든지 신청했던 걸 취소할 수 있도록.
2-6. 로그인을 하면 일정을 신청한 사람들의 명단을 볼 수 있지만,
2-7. 로그인하지 않은 게스트 모드에서는 명단을 조회할 수 없음.

2-8. 가고 싶은 가게도 신청할 수도 있지만, 먹고싶은 메뉴만 신청하는 기능도 추가.

2-9. 신청한 일정, 가게, 메뉴에 유저들이 '좋아요'와 댓글을 달 수 있음.

고민중인 것
- 2-4번 기능은 무료로 구현하기 어려움.
- 게스트 모드에서 일정에 참가신청하고 노쇼하는 걸 어떻게 막지?
- 링크만 넣기보다, 네이버/카카오 지도 API를 연동해 사이트에서 가게 위치를 띄워주는 기능도 고려. (일정 횟수까지는 무료)
 
## 채팅방 ##
3-1. 일정을 신청한 사람들끼리 채팅할 수 있는 채팅방
3-2. 카카오 오픈채팅방 링크를 자동 생성.



## 되는시간/안되는시간 ##
4-1. 유저의 되는시간 혹은 안되는시간을 시간표처럼 보여줘서 일정을 조율할 때 활용.
4-2. 되는시간과 안되는시간은 유저의 개인정보에서 언제든지 수정할 수 있음. 안되는 요일과 시간, 안되는 날짜를 입력할 수 있음.

4-3. 유저가 가게나 일정에 참가신청을 할 때 되는시간/안되는시간을 갱신할지 시스템이 물어봄. 유저가 수락할 경우 자신의 되는시간/안되는 시간을 갱신할 수 있음.
4-4. 일정을 신청한 유저가 일정 수 이상이 되면 유저들의 되는시간을 보여주는 페이지와 안되는 시간을 보여주는 페이지가 생김.

4-5. 다른 유저에게 되는시간/안되는시간을 공개할지 비공개할지 선택할 수 있음.

4-6. 되는시간과 안되는시간을 분리해서, 되는시간은 일정 추천에 사용하고 안되는시간은 일정 필터링에 사용.
 
## 필터 ##
5-1. 해산물 제외한 가게/일정만 보이게 하는 필터기능
5-2. 나의 되는시간에 예정된 일정만 보이게 하는 필터기능



## 리뷰 ## 
6-1. 리뷰는 일정 신청과 연계되어 갔다온 일정에 리뷰를 남길 수 있음. 사진, 동영상, 텍스트를 입력할 수 있음. 
6-2. 일정의 날짜가 자동으로 리뷰 위에 입력됨.
6-3. 지도에 갔다온 가게를 마커로 표시하고, 마커를 클릭하면 유저들이 작성한 리뷰를 볼 수 있음.

고민중인 것.
- 무료저장용량을 고려해서, 동영상은 YouTube(일부 공개)나 인스타그램에 올린 뒤 링크만 가져와서 임베드하는 방식을 고려.



## 사건사고 대응 ##
7-1. 사사게(사건사고게시판) 만들기.

7-2. 모임매너가 안좋았던 사람을 신고할 수 있음.
7-3. 신고가 많이 쌓인 유저는 밴 됨. 회원탈퇴.
 
## 시스템 ##
8-1. 가입할 때 시스템에 대한 설명 보여주기. 스킵가능. 
8-2. 나중에 까먹거나 궁금하면 원하는 기능에 대한 설명만 검색해서 찾아볼 수 있음.

8-3. 유저가 건의사항을 입력하는 페이지
8-4. 페이지에 건의글이 올라오면 다른 유저가 건의글에 '좋아요'나 댓글을 달 수 있음. 
8-5. 내 유저 ID는 사이트의 관리자권한을 가지고 유저의 건의사항을 '완료됨' 처리 할 수 있음.

8-6. PC와 모바일의 전용 UI를 각각 만들기.
 
## 사용할 기술 스택 ##
서버리스 스택 (Next.js + Supabase + Vercel)

프레임워크 
: Next.js
-	React를 기반으로하는 웹 개발 프레임워크.
-	자바스크립트 사용.
-	PC랑 모바일 UI를 모두 잡을 수 있음.
-	프론트엔드(화면)랑 백엔드(API)를 한 프로젝트 안에서 다 끝낼 수 있음.


데이터베이스 & 인증
: Supabase
-	로그인이나 DB, 채팅 기능을 무료로 제공.


호스팅/배포
: Vercel
-	제작한 사이트를 클릭 몇 번으로 실제 인터넷 주소(my-tasty-club.vercel.app 같이)로 바로 띄울 수 있음.
