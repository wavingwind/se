# 요구사항 정의서 #
## [ 맛집탐방 모임 관리 시스템 ] ##
# 1. 서론 #
## 1-1. 문서 목적 및 범위 ##
이 문서는 맛집탐방 모임 관리 웹사이트에서 각 요구사항이 무엇이지 조사하고, 정의하는 문서이다. <br/>
이 문서는 기능적, 비기능적, 인터페이스에서 요구되는 사항들을 정의한다. <br/>
<br/>

## 1-2. 프로젝트 개요 ##
### 1-2-1. 프로젝트 정의 ###
맛집탐방 일정을 세워 참가자들을 모집하고 링크를 통해 모집글에 쉽게 참가요청 할 수 있는 웹사이트를 제작한다. <br/>
<br/>

### 1-2-2. 주요 기능 설명 ###
1) 원하는 식당을 정해서 맛집탐방 참여자를 모집할 수 있는 '레이드' <br/>
2) 먹고싶은 메뉴를 제안하고, 좋아요를 눌러 관심도를 나타낼 수 있는 '먹고싶은거' <br/>
3) 자신의 프리한 시간을 올려 맛집탐방 일정에 데려가 달라는 글을 쓸 수 있는 '파티구직' <br/>
<br/>

# 2. 기능적 요구사항 #
### 로그인 기능 ###
사용자는 아이디, 비밀번호, 이메일 주소를 입력하여 회원가입을 할 수 있다. <br/>
>> 우선순위:중 <br/>

사용자는 아이디, 비밀번호를 입력하여 로그인한다. <br/>
>> 우선순위:중 <br/>

로그인 한 사용자는 '레이드' 모집 글을 올릴 수 있다. <br/>
>> 우선순위:상 <br/>
<br/>

### 맛집 레이드 기능 ###
'레이드' 모집 글을 올린 사용자는 '파티장'이 된다. '파티장'은 레이드할 식당과 시간을 작성하고 카카오톡 오픈채팅방 링크를 적을 수 있다. <br/>
>> 우선순위:상 <br/>

파티장은 참가신청을 해놓고 오픈채팅방에는 들어오지 않은 사용자를 강퇴(kick) 할 수 있다. <br/>
>> 우선순위:상 <br/>

사용자는 로그인하지 않아도 모집글이 올라온 레이드에 참가신청을 할 수 있다. <br/>
>> 우선순위:상 <br/>
<br/>

### 먹고싶은거 ###
사용자는 본인이 먹고 싶은 메뉴를 올리고, 다른 사람들이 올린 메뉴에 좋아요를 눌러 추천할 수 있다. <br/>
>> 우선순위:중 <br/>
<br/>

### 파티구직 ###
사용자는 자신이 맛집탐방에 참가하고 싶은 시간을 올려서 사람들에게 자신을 데려가달라고 할 수 있다. <br/>
>> 우선순위:중 <br/>
<br/>

### 프로필 사진 기능 ###
로그인한 사용자는 자신의 프로필 이미지를 추가할 수 있다. <br/>
>> 우선순위:하 <br/>
<br/>

### 리뷰 기능 ###
맛집 레이드에 참가한 사용자는 갔다온 식당에 대해 리뷰를 남길 수 있다. <br/>
>> 우선순위:하 <br/>
<br/>

# 3. 비기능적 요구사항 #
### 성능 요구사항 ###
데이터베이스는 실시간으로 업테이트 된다. <br/>
>> 우선순위:상 <br/>

페이지를 넘어갈 때 딜레이가 발생하면 버퍼링 이미지를 출력한다. <br/>
>> 우선순위:하 <br/>
<br/>

### 보안 요구사항 ###
시스템은 모든 사용자의 개인정보를 암호화하여 저장한다. <br/>
>> 우선순위:상 <br/>
<br/>

# 4. 인터페이스 요구사항 #
PC와 모바일 모두에서 사용할 수 있도록 UI를 만든다. <br/>
>> 우선순위:상 <br/>
